## 环境准备

### 1. 服务器要求
- **操作系统**: Linux (CentOS/Ubuntu/Debian)
- **架构**: x86_64 / ARM64
- **依赖**: Go 1.21+ (编译时), 无需运行时依赖 (静态编译)
- **端口**: 18318 (默认, 可配置)

### 2. 配置文件说明

| 文件 | 用途 | `auth-dir` |
|------|------|------------|
| `config.yaml` | 本地开发（不部署到服务器） | `./auths` |
| `config-277.yaml` | 227 服务器生产配置 | `/opt/cliproxy/auths` |

> ⚠️ 生产环境统一使用 `config-277.yaml`，`rsync` 时会排除本地 `config.yaml`。

### 3. 本地准备

#### 2.1 同步二进制文件到服务器
```bash
# 方式一: 使用 scp
scp -i "$HOME/.ssh/id_ed25519" ./cli-proxy root@192.168.15.227:/opt/cliproxy/

# 方式二: 同步整个目录
rsync -avz --exclude '.git' --exclude 'worktrees' --exclude 'config.yaml' --exclude 'config-277.yaml' --exclude 'config.yaml.*' --exclude 'logs' --exclude 'auths' \
  -e "ssh -i $HOME/.ssh/id_ed25519" \
  ./ root@192.168.15.227:/opt/cliproxy/
```

#### 2.2 确保配置文件存在
```bash
# 检查远程服务器配置文件
ssh -i "$HOME/.ssh/id_ed25519" root@192.168.15.227 "ls -la /opt/cliproxy/config-277.yaml"
```

**config-277.yaml 关键配置项说明:**
```yaml
host: "0.0.0.0"
port: 18318
auth-dir: "/opt/cliproxy/auths"      # 认证文件目录
proxy-url: "http://192.168.15.161:7891"  # 上游代理
api-keys:
  - your-api-key-here

# 远程管理配置
remote-management:
  allow-remote: true
  secret-key: "$2a$10$..."          # bcrypt hash 管理密钥

# 路由策略
routing:
  strategy: "round-robin"           # 轮询/权重
```

## 部署流程

### 快速启动 (推荐)

```bash
# 一条命令完成重启
ssh -i "$HOME/.ssh/id_ed25519" -o IdentitiesOnly=yes -o StrictHostKeyChecking=no root@192.168.15.227 \
  "pkill -f cli-proxy 2>/dev/null; cd /opt/cliproxy && nohup ./cli-proxy -config config-277.yaml > /tmp/cliproxy.log 2>&1 &"
```

### 详细步骤

#### 1. 停止旧进程
```bash
# 先记录发布前 PID（用于发布后核验）
ssh root@192.168.15.227 "pgrep -f './cli-proxy -config config-277.yaml' | head -n 1"

# 停止旧进程
ssh root@192.168.15.227 "pkill -f cli-proxy 2>/dev/null || true"
```

#### 2. 启动服务
```bash
# 后台启动
ssh root@192.168.15.227 "cd /opt/cliproxy && nohup ./cli-proxy -config config-277.yaml > /tmp/cliproxy.log 2>&1 &"

# 或前台启动查看输出 (调试用)
ssh root@192.168.15.227 "cd /opt/cliproxy && ./cli-proxy -config config-277.yaml"
```

#### 3. 验证进程
```bash
# 检查进程是否存在
ssh root@192.168.15.227 "ps aux | grep cli-proxy | grep -v grep"

# 核验发布前后 PID 是否已切换（示例）
# old_pid 为停止前记录的 PID
# new_pid 为重启后最新 PID
old_pid="<停止前PID>"
new_pid=$(ssh root@192.168.15.227 "pgrep -f './cli-proxy -config config-277.yaml' | head -n 1")
[ -n "$new_pid" ] && [ "$new_pid" != "$old_pid" ] && echo "PID核验通过" || echo "PID核验失败"

# 检查端口是否监听
ssh root@192.168.15.227 "ss -tlnp | grep 18318"
```

> 发布核验要求：必须同时满足 `new_pid` 存在且 `new_pid != old_pid`，否则不能判定发布成功。

## 常见问题

### Q1: 启动后无日志输出
```bash
# 检查日志文件是否创建
ssh root@192.168.15.227 "ls -la /tmp/cliproxy.log"

# 如果文件不存在，手动创建目录并重新启动
ssh root@192.168.15.227 "cd /opt/cliproxy && ./cli-proxy -config config-277.yaml 2>&1 | tee /tmp/cliproxy.log"
```

### Q2: 端口被占用
```bash
# 查找占用进程
ssh root@192.168.15.227 "ss -tlnp | grep 18318"

# 强制终止
ssh root@192.168.15.227 "kill -9 <PID>"
```

### Q3: 管理 API 返回 "invalid management key"
- 检查 config-277.yaml 中 `remote-management.secret-key` 是否正确
- 使用 bcrypt 加密的密钥，或在配置中设置明文密钥 (启动时会自动 hash)

### Q4: 连接上游代理失败
```bash
# 检查上游代理连通性
ssh root@192.168.15.227 "curl -x http://192.168.15.161:7891 https://api.openai.com/v1/models"

# 检查 config-277.yaml 中 proxy-url 配置
```

### Q5: 认证文件加载失败
```bash
# 检查 auths 目录权限和文件
ssh root@192.168.15.227 "ls -la /opt/cliproxy/auths/"

# 检查认证文件格式
ssh root@192.168.15.227 "cat /opt/cliproxy/auths/*.json | head -50"
```

## 回滚方案

### 1. 快速回滚到上一版本
```bash
# 停止当前版本
ssh root@192.168.15.227 "pkill -f cli-proxy"

# 恢复备份
ssh root@192.168.15.227 "cd /opt/cliproxy && cp cli-proxy.bak.* cli-proxy"

# 重启
ssh root@192.168.15.227 "cd /opt/cliproxy && nohup ./cli-proxy -config config-277.yaml > /tmp/cliproxy.log 2>&1 &"
```

### 2. 使用 start.sh 管理 (推荐)
```bash
# 在服务器上执行
ssh root@192.168.15.227 "cd /opt/cliproxy && ./start.sh stop"
ssh root@192.168.15.227 "cd /opt/cliproxy && ./start.sh start"
```

### 3. 版本备份脚本
```bash
#!/bin/bash
# 部署前备份
DATE=$(date +%Y%m%d_%H%M%S)
ssh root@192.168.15.227 "cp /opt/cliproxy/cli-proxy /opt/cliproxy/cli-proxy.bak.\$DATE"
```
