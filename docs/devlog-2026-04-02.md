## 二、iOS token 与 Codex token 的差异

排查发现项目原有实现只支持 Codex CLI OAuth 流，iOS ChatGPT App 导出的 token 有以下关键差异：

| 对比项 | Codex CLI | iOS ChatGPT App |
|---|---|---|
| `client_id` | `app_EMoamEEZ73f0CkXaXp7hrann` | `app_LlGpXReQgckcGGUo2JrYvtJK` |
| `redirect_uri` | `http://localhost:1455/auth/callback` | `com.openai.chat://auth0.openai.com/ios/com.openai.chat/callback` |
| 请求编码 | `application/x-www-form-urlencoded` | `application/json` |
| id_token `aud` | Codex client_id | iOS client_id |
| id_token auth 字段 | 含 `chatgpt_account_id`、套餐信息等 | 只有 `user_id` |

iOS refresh_token 必须用 iOS client_id 才能续期，用错 client_id 会返回 `refresh_token_expired`。

## 三、代码修改

### 3.1 `internal/auth/codex/openai_auth.go`

- 新增常量 `IOSClientID`、`IOSRedirectURI`
- 新增 `RefreshTokensIOS()`：使用 JSON 请求体 + iOS client_id + redirect_uri 发起续期
- 新增 `IsIOSToken(idToken)`：解析 id_token 的 `aud` 字段，判断是否为 iOS token
- `RefreshTokens()` 和 `ExchangeCodeForTokens()` 加入 `log.Debugf` 记录完整响应体

### 3.2 `internal/auth/codex/jwt_parser.go`

- `GetAccountID()` 增加兜底逻辑：`chatgpt_account_id` 为空时返回 `user_id`
- 修复 iOS token 在管理面板显示"Codex 凭证缺少 ChatGPT 账号 ID"的报错

### 3.3 `internal/api/handlers/management/auth_files.go`

- `chatgpt_account_id` 的取值改为调用 `claims.GetAccountID()`，与上面兜底逻辑保持一致

### 3.4 `internal/runtime/executor/codex_executor.go`

- `Refresh()` 增加读取 `auth.Metadata["id_token"]`

## 四、批量导入脚本

输入格式（`----` 分隔，取首段 email 和末段 refresh_token）：
```
email----忽略----忽略----rt_xxx
```

编写了 `scripts/import_codex_ios_tokens.py`，支持命令行参数：

```bash
python3 scripts/import_codex_ios_tokens.py codex.txt
python3 scripts/import_codex_ios_tokens.py codex.txt --plan plus --auth-dir /home/iec/deploy/auths
```

本次会话共处理 3 批次，合计 **67 个** iOS token（17 + 15 + 35），全部成功生成 auth JSON 文件到 `~/deploy/auths/`。

## 五、auth JSON 文件结构

```json
{
  "access_token":  "eyJ...",
  "account_id":    "user-C0AlW40Fl1ViVpdGuxTnoJck",
  "disabled":      false,
  "email":         "alice@hotmail.com",
  "expired":       "2026-04-12T23:48:50+08:00",
  "id_token":      "eyJ...",
  "last_refresh":  "2026-04-02T23:48:50+08:00",
  "refresh_token": "rt_s-6Niy...",
  "type":          "codex"
}
```

iOS token 的 `account_id` 为 `user-` 前缀格式（来自 `user_id`），Codex token 为 UUID 格式（来自 `chatgpt_account_id`）。
